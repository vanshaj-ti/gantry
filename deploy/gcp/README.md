# Deploying gantry to GCP against edupaid

Runs gantry as two always-on Docker containers on one GCE VM:
- `gantry-advance` — ticks `advance --all` against a persistent edupaid clone.
- `gantry-linear` — receives Linear webhooks, classifies tickets, creates runs.

Project: `aristotle-436708` (confirm this is correct before running anything
here — override with `PROJECT_ID=... ./01-setup-vm.sh` etc if not).

## Order of operations

```
01-setup-vm.sh       # create the GCE VM, firewall rule, service account
02-secrets.sh         # create Secret Manager secrets (prompts for real values)
03-deploy.sh          # SSH in, clone edupaid, build image, start both containers
```

Each script is idempotent-ish (safe to re-run) and prints what it's about to
do before doing it. None of them run automatically — read each one before
running.

## Prerequisites you provide (not generated by these scripts)

- A fine-grained GitHub PAT scoped to the edupaid repo only
  (contents: read/write, pull requests: read/write).
- This org routes Claude Code through TrueFoundry's gateway (Maat), not raw
  Anthropic — you need `ANTHROPIC_BASE_URL` (the gateway URL, e.g.
  `https://tfy.promptlens.trilogy.com`) and `ANTHROPIC_AUTH_TOKEN` (your
  personal TrueFoundry API key from Maat → Profile → User key → Reveal), not
  a plain `ANTHROPIC_API_KEY`. Both get passed into the container so
  headless Claude Code auth works with no interactive `/login` on the VM.
- edupaid's Linear team id (`GANTRY_LINEAR_TEAM_ID`) and a Linear personal
  API key or OAuth token with issue read/write + comment + label scopes
  (`GANTRY_LINEAR_API_KEY`).
- `GANTRY_LINEAR_WEBHOOK_SECRET` — you choose this value now; Linear asks
  for it again when you register the webhook in step 4 (`04-register-webhook.md`),
  so the two must match — Linear does NOT generate this for you.

## After 03-deploy.sh

Follow `04-register-webhook.md` to point a real Linear webhook at the VM's
public endpoint, then `05-verify.md` to run the real Linear-ticket-to-PR test.

## Human-gated stages: answered via Linear comments

Linear is the only human channel for this deployment — no Telegram. When a
run hits a human-gated stage (e.g. `investigation_complete`, awaiting
approval), gantry's comment on the issue is where you reply:
- Reply `approve` (or `yes`/`y`) to advance to the next stage.
- Reply with anything else to send that text back as feedback, which
  resumes the stage with your answer.

This is handled deterministically by `gantry/linear.py`'s
`handle_comment_created` — a plain status lookup + string match (same logic
`gantry listen`'s Telegram path already used), not an agent call. No manual
`docker exec ... gantry approve` needed for normal operation; that remains
available as a fallback if the webhook is down.
